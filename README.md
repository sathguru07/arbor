## What is Arbor?

Most AI coding assistants treat your codebase like a bag of text. They search for keywords, embed chunks into vectors, and hope for the best. The result? Hallucinated connections, missing context, and refactors that break everything downstream.

**Arbor thinks differently.**

We parse your code into an Abstract Syntax Tree using [Tree-sitter](https://tree-sitter.github.io/), then build a living graph where every function, class, and variable is a **node**, and every import, call, and implementation is an **edge**. When an AI asks "where is authentication handled?", Arbor doesn't grep for "auth" ‚Äî it traces the call graph to find the actual service that initiates the flow.

```
Traditional RAG:         Arbor:
                         
"auth" ‚Üí 47 results      "auth" ‚Üí AuthController
                                  ‚îú‚îÄ‚îÄ validates via ‚Üí TokenMiddleware  
                                  ‚îú‚îÄ‚îÄ queries ‚Üí UserRepository
                                  ‚îî‚îÄ‚îÄ emits ‚Üí AuthEvent
```

## Quick Start

```bash
# Install Arbor CLI
cargo install arbor-cli

# Initialize in your project
cd your-project
arbor init

# Index your codebase (typically < 2 seconds)
arbor index

# Start the context sidecar
arbor serve
```

That's it. Your IDE or AI agent can now connect to `ws://localhost:7432` and query the graph.

## Features

### üå≤ AST-Graph Intelligence

Every code entity becomes a queryable node. Arbor understands scope, shadowing, and namespace isolation ‚Äî so when you ask for context, you get the exact logical block, not keyword-matched noise.

### ‚ö° Sub-100ms Incremental Sync

Arbor watches your files and re-parses only the changed AST nodes. In a 100k-line monorepo, saving a file triggers a ~15ms update. You'll never notice it running.

### üîç Blast Radius Analysis

Refactoring a function? Arbor traces every caller, every consumer, every downstream dependency. See the full impact before you break production.

### üìä Semantic Ranking

Not all code is equal. Arbor ranks nodes by "centrality" ‚Äî a function called by 50 others is more architecturally significant than a one-off utility. Context windows get the important stuff first.

### üé® Logic Forest Visualizer

The optional desktop app renders your codebase as an interactive 3D graph. Custom shaders create bloom and glow effects as you navigate. It's not just useful ‚Äî it's beautiful.

## The Protocol

The Arbor Protocol is a simple JSON-RPC interface over WebSocket. Here's what your AI agent can ask:

```json
// Find the architectural root for a concept
{
  "method": "discover",
  "params": { "query": "user authentication" }
}

// Get the blast radius for a function
{
  "method": "impact",
  "params": { "node": "UserService.validateToken" }
}

// Retrieve ranked context for a task
{
  "method": "context",
  "params": { 
    "task": "refactor the payment flow",
    "maxTokens": 8000
  }
}
```

See [docs/PROTOCOL.md](docs/PROTOCOL.md) for the full specification.

## Performance

We obsess over speed because slow tools don't get used.
